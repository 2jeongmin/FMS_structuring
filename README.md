# 프롬프트 템플릿

**평가 기준 버전**: 수정된 평가항목 및 세부배점 (2025.11.19 최종)  
**총점**: 100점 (A항목 80점 + B항목 20점)

## 사용 방법

### 1단계: 파일 구조
```
📁 프로젝트 폴더/
├── SYSTEM_PROMPT.md          (시스템 역할 정의)
├── EVALUATION_CRITERIA.md    (평가 기준 상세)
├── schema.json                (출력 형식 정의)
├── 데이터파일.csv             (시설물 데이터)
└── 평가기준.pdf               (공식 지침서)
```

### 2단계: 프롬프트 구성
아래 템플릿을 사용하여 프롬프트를 작성합니다.

---

## 기본 프롬프트 템플릿

```markdown
# 역할 및 평가 기준
[SYSTEM_PROMPT.md 파일 내용 참조]
[EVALUATION_CRITERIA.md 파일 내용 참조]

# 출력 형식
[schema.json 파일 내용 참조]

# 평가 대상
시설물명: {시설물명}

# 데이터 소스
1. 업로드된 CSV/Excel 파일: {파일명}
2. 평가기준 PDF: {PDF파일명}
3. 필요시 웹 검색

# 요청사항
위 평가 기준에 따라 {시설물명}에 대한 전담시설물 평가를 수행하고, 
schema.json 형식으로 결과를 출력해주세요.

평가 시 다음 사항을 준수해주세요:
1. 모든 배점에 대한 명확한 근거 제시
2. 데이터 출처 명시
3. 불확실한 정보는 명시적으로 표기
4. 실무 활용 가능한 권고사항 작성
5. A항목과 B항목 점수를 명확히 구분하여 제시
```

---

## 프롬프트 작성 요령

### DO
1. **명확한 구조**: 섹션을 명확히 구분
2. **구체적 요청**: "평가해줘" 보다 "schema.json 형식으로 평가해줘"
3. **예시 포함**: Few-shot으로 기대하는 출력 형태 제시
4. **검증 요청**: "배점 근거를 명시해줘"
5. **파일 참조**: "EVALUATION_CRITERIA.md를 참조하여..."

### DON'T
1. ~~모호한 표현~~: "대충 평가해줘"
2. ~~파일 없이 요청~~: 평가 기준 없이 "점수 매겨줘"
3. ~~일관성 없음~~: 매번 다른 기준으로 요청
4. ~~검증 누락~~: 결과 확인 없이 사용
5. ~~출처 생략~~: 어떤 데이터 기반인지 불명확

## 주요 변경사항 (2025.11.19 최종)

### 평가 체계
- **총점**: 100점 (A항목 80점 + B항목 20점)
- **배점 구성**: 

| 대분류 | 중분류 | 배점 |
|--------|--------|------|
| **A. 안전관리 필요성** | 1. 기능성 | 15점 (상대평가) |
| | 2. 위험성 | 40점 (절대평가) |
| | 3. 공공성 | 10점 (절대평가) |
| | 4. 보안성 | 5점 (절대평가) |
| | 5. 신기술성 | 10점 (절대평가) |
| **A항목 소계** | | **80점** |
| **B. 관리주체 안전관리 역량** | 1. 안전의무 이행 | 10점 (절대평가) |
| | 2. 안전성과 관리의 전문성 | 10점 (절대평가) |
| **B항목 소계** | | **20점** |
| **총계** | | **100점** |

### A항목 상세 변경

#### 1. 기능성 (15점)
- **평가방식**: 상대평가 (순위 기반)
- **평가산식**: (규모1 순위/총 개소수)×7.5 + (규모2 순위/총 개소수)×7.5

#### 2. 위험성 (40점) - 3개 세부항목
| 세부항목 | 배점 | 주요 기준 |
|----------|------|-----------|
| 2-1. 재난발생 가능성 | 15점 | RCP6.0 & SSP2-4.5 시나리오 |
| 2-2. 안전등급 변화 추이 | 15점 | 5년/10년 이내 2단계 변화 |
| 2-3. 반복적인 중대결함 발생 이력 | 10점 | 3회/7회/10회 기준 |

#### 3. 공공성 (10점)
- **평가기준**: 인구수 + 인구밀도
- **기준값**: 
  - 상위: 인구 5백만 & 인구밀도 4,500명/km²
  - 하위: 인구 2.5백만 & 인구밀도 3,000명/km²

#### 4. 보안성 (5점)
- 보안시설물 & 국가핵심기반시설 여부

#### 5. 신기술성 (10점)
- 신기술 적용 가능성 & 신형식/신공법 시설물 여부

### B항목 상세 (20점)

#### 1. 안전의무 이행 (10점) - 2개 세부항목
| 세부항목 | 배점 | 평가 기준 |
|----------|------|-----------|
| 2-1. 법정 안전점검 및 성능평가 이행 | 5점 | 최근 5년간 제출 이행도 |
| 2-2. 시설물 관리계획 적기 제출 | 5점 | 최근 5년간 제출 이행도 |

**배점 기준 (각 5점)**:
- 미이행 또는 미제출 1건 이상: 5점
- 지연 제출 1건 이상: 3점
- 기한 내 제출: 0점

#### 2. 안전성과 관리의 전문성 (10점) - 2개 세부항목
| 세부항목 | 배점 | 평가 기준 |
|----------|------|-----------|
| 3-1. 점검 및 진단보고서 평가결과의 부적정 | 5점 | 부적정률 0% / 0~5% / 5% 초과 |
| 3-2. 점검 및 진단보고서 평가결과의 시정 | 5점 | 시정률 100% / 95~100% / 95% 미만 |

**중요**: B항목은 점수가 높을수록 관리주체 역량이 낮음을 의미 (전담시설물 지정 필요성 증가)

## 필요 데이터 목록

### A항목 평가 데이터
1. **기능성**
   - 시설물 규모 (총연장, 최대경간장, 연면적, 층수, 저수용량 등)
   
2. **위험성**
   - 기후변화 시나리오 데이터 (RCP6.0, SSP2-4.5)
   - 안전등급 변화 이력 (최근 10년)
   - 중대결함 발생 횟수 및 상세 내용
   
3. **공공성**
   - 시/도 단위 인구수
   - 시/군/구 단위 인구밀도
   
4. **보안성**
   - 보안시설물 여부
   - 국가핵심기반시설 지정 여부
   
5. **신기술성**
   - 신기술 적용 가능성
   - 신형식/신공법 적용 여부

### B항목 평가 데이터
1. **안전의무 이행**
   - 점검/진단/성능평가 보고서 제출 이력 (최근 5년)
     - 기한 내 제출 건수
     - 지연 제출 건수
     - 미제출 또는 미이행 건수
   - 시설물관리계획서 제출 이력 (최근 5년)
     - 기한 내 제출 건수
     - 지연 제출 건수
     - 미제출 건수
   
2. **안전성과 관리의 전문성**
   - 보고서 평가 결과 (최근 5년)
     - 전체 보고서 수
     - 부적정 보고서 수
     - 부적정률 (%)
   - 보고서 시정 현황 (최근 5년)
     - 시정 완료 보고서 수
     - 미시정 보고서 수
     - 시정률 (%)

## 데이터 부족 시 처리 방침

| 데이터 종류 | 처리 방법 |
|-------------|-----------|
| **필수 데이터 누락** | 명시적 표시 및 0점 처리 |
| **선택 데이터 누락** | 보수적 평가 (낮은 등급 적용) |
| **불명확한 데이터** | 합리적 추정 후 근거 명시 |
| **경계값** | 상위 등급 적용 |

## 평가 결과 활용

### 점수 해석
- **A항목 (80점)**: 시설물 자체의 전담관리 필요성
  - 높을수록 전담관리 필요성 高
  
- **B항목 (20점)**: 관리주체의 안전관리 역량 부족도
  - 높을수록 관리주체 역량 低 → 전담관리 필요성 高

### 종합 평가
- **총점 (100점)**: A항목 + B항목
  - 높을수록 전담시설물 지정 필요성 증가

---

## 주의사항

1. **B항목의 특수성**: 일반적인 평가와 달리, 점수가 높을수록 관리주체의 역량이 낮음을 의미
2. **상대평가 vs 절대평가**: 기능성은 상대평가이므로 후보군 구성에 따라 점수 변동 가능
3. **데이터 품질**: 평가 결과의 정확도는 입력 데이터의 품질에 직접적으로 의존
4. **최근 5년 데이터**: B항목 평가는 최근 5년간의 관리주체 실적 기반
5. **기후변화 데이터**: 미래 예측값이므로 불확실성 존재
