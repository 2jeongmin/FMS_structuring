# 프롬프트 템플릿

## 사용 방법

### 1단계: 파일 구조
```
📁 프로젝트 폴더/
├── SYSTEM_PROMPT.md          (시스템 역할 정의)
├── EVALUATION_CRITERIA.md    (평가 기준 상세)
├── schema.json                (출력 형식 정의)
├── 데이터파일.csv             (시설물 데이터)
└── 평가기준.pdf               (공식 지침서)
```

### 2단계: 프롬프트 구성
아래 템플릿을 사용하여 프롬프트를 작성합니다.

---

## 기본 프롬프트 템플릿

```markdown
# 역할 및 평가 기준
[SYSTEM_PROMPT.md 파일 내용 참조]
[EVALUATION_CRITERIA.md 파일 내용 참조]

# 출력 형식
[schema.json 파일 내용 참조]

# 평가 대상
시설물명: {시설물명}

# 데이터 소스
1. 업로드된 CSV/Excel 파일: {파일명}
2. 평가기준 PDF: {PDF파일명}
3. 필요시 웹 검색

# 요청사항
위 평가 기준에 따라 {시설물명}에 대한 전담시설물 평가를 수행하고, 
schema.json 형식으로 결과를 출력해주세요.

평가 시 다음 사항을 준수해주세요:
1. 모든 배점에 대한 명확한 근거 제시
2. 데이터 출처 명시
3. 불확실한 정보는 명시적으로 표기
4. 실무 활용 가능한 권고사항 작성
```


## 프롬프트 작성 요령

### DO
1. **명확한 구조**: 섹션을 명확히 구분
2. **구체적 요청**: "평가해줘" 보다 "schema.json 형식으로 평가해줘"
3. **예시 포함**: Few-shot으로 기대하는 출력 형태 제시
4. **검증 요청**: "배점 근거를 명시해줘"
5. **파일 참조**: "EVALUATION_CRITERIA.md를 참조하여..."

### DON'T
1. ~~모호한 표현~~: "대충 평가해줘"
2. ~~파일 없이 요청~~: 평가 기준 없이 "점수 매겨줘"
3. ~~일관성 없음~~: 매번 다른 기준으로 요청
4. ~~검증 누락~~: 결과 확인 없이 사용
5. ~~출처 생략~~: 어떤 데이터 기반인지 불명확
